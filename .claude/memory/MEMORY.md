# JRV CRM — Project Memory

## Architecture
- **Framework:** Next.js 14 (App Router)
- **Database:** Supabase (leads table)
- **Auth:** Supabase Auth
- **Deploy:** Vercel → https://lead-crm-wine.vercel.app
- **GitHub:** github.com/JRVGLOBALSERVICES/jrv-crm

## Key Features
- Dashboard with stats, filters, pagination, sorting
- Lead detail page with status management, enrichment tabs, competitors, friction reviews
- Proposal page generated by Lead Architect
- JP: Add via Google Maps URL → auto-enrichment pipeline

## Enrichment Pipeline
- **Thor cron** (daily 10AM): scrapes GBP + Places API → inserts leads → spawns Marvel sub-agents
- **Maps Lead Enrichment cron** (every 15min): checks for unenriched maps leads → scrapes GBP via Playwright + Places API fallback → spawns Marvel sub-agents
- **Lead Architect cron** (hourly): generates proposals for replied/pitched leads
- Scripts: `scripts/enrich-from-maps.cjs` (standalone Playwright + Places API), `scripts/pipeline.mjs` (insert, store-gbp, store-enrichment)

## Add Lead Flow
1. User pastes Google Maps URL in `/leads/new`
2. Business name auto-extracted from URL
3. Lead created with `google_maps_url` and status `new`
4. Cron picks up within 15 min → scrapes GBP data → spawns Marvel sub-agent
5. Sub-agent generates full enrichment JSON (market intel, digital audit, tech scope, pitch deck)
