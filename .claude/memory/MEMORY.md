# Lead CRM — Project Memory

**Lead management system for JRV Systems. Next.js + Supabase + Vercel.**

## Architecture
- **Frontend**: Next.js 14 (Pages Router + Client Components)
- **Database**: Supabase (Postgres)
- **Deployment**: Vercel
- **Lead Generation**: Thor cron job → Pipeline scripts → Supabase
- **Proposals**: Generated by Lead Architect tool → Stored in `notes.proposal_content`

## Key Files

| File | Purpose |
|------|---------|
| `src/app/page.tsx` | Dashboard — lists all leads from Supabase |
| `src/app/leads/[id]/page.tsx` | Lead detail — status, notes, GBP data, enriched fields |
| `src/app/leads/[id]/proposal/page.tsx` | Proposal viewer — renders from `notes.proposal_content` |
| `src/app/leads/new/page.tsx` | Add new lead form |
| `src/app/api/leads/route.ts` | API — GET all, POST new (reads/writes Supabase) |
| `src/app/api/leads/[id]/route.ts` | API — GET/PATCH/DELETE single lead |
| `src/lib/supabase.ts` | Supabase client (anon + service role) |
| `src/lib/types.ts` | TypeScript types for Lead |

## CRM Features
- **Dashboard**: Stats cards (Total/New/Contacted/Replied/Pitched/Closed/Lost)
- **Lead Detail**: Status management, notes editing, GBP data display (address, hours, rating, enriched_by)
- **Proposal Page**: `/leads/[id]/proposal` — renders proposal markdown from Supabase; shows "No Proposal Yet" if none
- **Enrichment Tabs**: Enriched data shown in lead detail (enriched_by, market_intel, digital_audit)

## Data Flow
1. **Thor cron** (daily 10AM) → Playwright MCP → browses Google Maps
2. Thor extracts: name, category, rating, reviews, phone, maps URL, address, hours, gbp_rating, is_zero_state, website_url
3. Inserted via `pipeline.mjs insert` → Supabase `leads` table
4. Marvel sub-agents research each lead → `pipeline.mjs store-enrichment` → updates `notes`, `enriched_by`, `competitors`
5. Status updates via CRM UI
6. When status=replied/pitched → Lead Architect cron (hourly) generates proposal → stores in `notes.proposal_content`

## Database Schema (`leads` table)
Key columns beyond basic CRM fields:
- `enriched_by` — Marvel sub-agent name
- `gbp_rating` — Numeric rating from Google Maps
- `is_zero_state` — Boolean (no real website)
- `website_url` — URL of existing site (if any)
- `hours` — Business hours from Google Maps
- `address` — Full address from Google Maps
- `enrichment_complete` — Boolean
- `notes` — JSON blob: enrichment dossier + proposal_content + metadata

## Enrichment
- Template: `scripts/enrichment-template.txt`
- Sub-agents output: full_dossier, market_intel, digital_audit, tech_scope, pitch_deck
- GBP data (address, hours, gbp_rating, is_zero_state) passed from Thor to sub-agents

## External Tools
- **Playwright MCP**: Browser automation for Google Maps (headless, via mcporter)
- **Pipeline**: `scripts/pipeline.mjs` — insert, store-enrichment, enrich, blacklist
- **Lead Architect**: `mcp-tools/lead-architect.mjs` — generates proposal slides
